当您准备从沙箱迁移到生产环境时，使用此页面。

当您需要以下内容时，从这里开始：

* 安全地从沙箱切换到生产环境
* 了解上线时会发生什么变化
* 监控首批生产订单和 Webhook 事件

### 常见问题

#### 何时可以生成生产凭证？

仅在 UAT 通过且您的客户经理将账户切换为 `LIVE` 之后。

仅通过 UAT 批准并不能解锁生产凭证。

#### 上线后应立即监控什么？

密切监控首批生产订单、支付结果、出票跟进和 Webhook 投递。

在正常生产流量增加之前，运行受控的冒烟测试。

### 本阶段的目标

将集成从沙箱迁移到生产环境。

这是整个上线流程中的最后一步。

### 如果以下情况尚未完成，请勿开始上线

当以下任何一项仍未完成时，请等待再切换环境：

* UAT 未获批准
* 账户未切换为 `LIVE`
* 生产凭证尚未生成
* 团队尚未准备好监控首批生产流量

### 何时开始

在以下条件满足后开始此步骤：

* 所需的 UAT 工作已完成
* 您的预订流程稳定
* Webhook 处理已就绪
* 您的团队可以监控首批生产订单

在上线检查清单准备就绪之前，不要开始生产设置。

仅当 UAT 通过且您的客户经理将账户切换为 `LIVE` 状态后，您才能生成生产凭证。

### 上线前应准备好什么？

确保 UAT 已获批准、预订流程稳定、Webhook 处理已就绪，并且团队可以从第一个订单开始监控生产交易。

### 生产环境中会变化什么

将以下项目从沙箱迁移到生产环境：

* 凭证
* 端点
* 支付处理
* 运营监控

请求头规则保持不变。

生产环境中的预订和支付是真实交易。

在生产环境中，从 ATRIP 的 `My Profile` → `Company Information` 获取生产 API 基础 URL。

上线后不要重复使用沙箱基础 URL。

生产环境为 `搜索` 使用一个基础 URL，为所有其他交易 API 使用另一个基础 URL。

### 最佳实践

在受控发布中切换凭证和端点。

在扩大流量之前，端到端验证首个生产流程。

### 首次冒烟测试时应检查什么

在首次生产运行时确认以下项目：

* 使用生产凭证进行身份验证成功
* 每个 API 组使用正确的生产基础 URL
* 使用预期的生产方式完成支付
* 订单跟进和 Webhook 投递按预期工作

### 生成生产凭证

您的账户进入 `LIVE` 状态后，在 ATRIP 中生成生产客户端 ID 和客户端密钥。

在同一生产设置流程中设置您的 IP 白名单。

在您的服务器配置中替换沙箱凭证和沙箱端点。

使用生产环境的 `搜索` 基础 URL 处理搜索流量。

使用另一个生产基础 URL 处理所有其余的交易 API。

### 完成此阶段的标准

您在生产环境中已具备以下所有条件：

* 可用的生产凭证
* 账户状态已更新为 `LIVE`
* 正确的生产端点
* 成功的冒烟测试结果
* 已监控的首批生产订单和 Webhook

### 本阶段的产出

* 生产就绪环境
* 生产预订能力
* 已完成的上线后初步验证

### 重要说明

* 不要在沙箱环境中使用生产卡。
* 不要使用沙箱运价进行生产验证。
* 在服务端保留生产凭证。
* 确认您的生产 IP 白名单保持最新。
* 仅通过 UAT 并不能解锁生产凭证。您的客户经理必须将账户切换为 `LIVE`。

### 如果首次生产运行失败

使用此恢复顺序：

1. 确认账户状态为 `LIVE`。
2. 确认生产客户端 ID 和客户端密钥正确。
3. 确认 `搜索` 和其他交易 API 使用了正确的生产基础 URL。
4. 确认生产 IP 白名单包含活跃的源 IP。
5. 仅在错误配置得到纠正后重试。

### 下一步是什么？

首次生产冒烟测试成功后，持续监控早期生产流量，直到订单状态、支付处理和 Webhook 行为稳定。

### 下一步

上线后，持续监控早期生产流量，并将任何预订或支付问题快速引导至相关的故障排除页面。
