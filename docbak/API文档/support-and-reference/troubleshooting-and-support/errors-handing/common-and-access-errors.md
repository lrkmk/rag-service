当请求在业务处理开始前失败时，使用本页面。

当你需要以下内容时，从这里开始：

* 了解为什么请求在业务流程开始前失败
* 检查凭证、JSON 格式或访问范围
* 决定是修复请求还是重试一次

### 常见问题

#### 哪些常见的访问错误通常需要先修复请求？

诸如 `100`、`901` 和 `902` 等代码通常意味着必填字段、请求格式或权限范围不正确。

在重试前修复请求。

#### 哪些常见的访问错误可以重试一次？

诸如 `9999` 或 `-1` 等代码在问题看起来是临时性的情况下可以重试一次。

如果相同的失败重复出现，捕获请求上下文并升级。

### 常见代码

#### `100` 缺少必需的请求数据

缺少必填字段。

**操作**

* 检查必需的参数
* 重新构建请求体
* 验证后重试

#### `900` 未授权访问

凭证错误、账户状态无效，或请求的目标数据超出您的范围。

**操作**

* 检查 `x-atlas-client-id`
* 检查 `x-atlas-client-secret`
* 确认账户是活跃的

#### `901` 非法请求数据

请求格式无效。

**操作**

* 发送 JSON
* 检查字段类型和嵌套
* 在重试前验证必需的结构

#### `902` 访问被拒绝

由于权限或账户范围，请求被阻止。

**操作**

* 重新检查凭证
* 确认账户可以访问此功能
* 如果请求应该被允许，请联系 Atlas

#### `9999` 系统错误

意外的平台侧故障。

**操作**

* 重试一次
* 如果重复出现，捕获请求 ID 并升级

#### `-1` 通用错误

当没有返回更具体的代码时使用的回退错误。

**操作**

* 检查请求上下文
* 如果是临时性问题则重试
* 如果重复出现则升级

### 快速重试指南

#### 先修复请求

示例：

* `100`
* `901`
* `902`

#### 重试一次，如果重复则升级

示例：

* `9999`
* `-1`

### 推荐的故障排除顺序

1. 确认凭证和账户状态
2. 验证 JSON 格式和必填字段
3. 临时性失败重试一次
4. 如果错误重复出现，附带请求上下文升级

### 后续步骤

访问或请求问题修复后，返回您尝试使用的工作流页面并从那里继续。
