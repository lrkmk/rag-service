使用本部分进行 Atlas API 入门引导以及从沙箱到上线的完整实施生命周期。

当您需要以下内容时，从这里开始：

* 获取访问权限和凭证
* 按正确顺序构建集成
* 准备 UAT 验证材料
* 安全完成生产环境切换

### 集成指南包含什么

使用此部分进行：

* 入门引导和访问设置
* 沙箱实施
* 验证和上线准备
* 集成阶段设置任务

当您需要选择合适的预订流程或产品能力时，请使用[产品指南](/api-wen-dang/product-guides.md)。

当您需要精确的端点字段和架构时，请使用[API 参考](/api-wen-dang/api-reference.md)。

1. [快速入门](/api-wen-dang/readme-1/quick-start.md)

### 常见起点

#### 新的 Atlas API 集成

从[快速入门](/api-wen-dang/readme-1/quick-start.md)开始。

然后前往[沙箱访问](/api-wen-dang/readme-1/making-requests.md)和[沙箱开发](/api-wen-dang/readme-1/sandbox-development.md)。

#### UAT 和上线准备

使用[UAT 验证](/api-wen-dang/readme-1/uat-submission-guide.md)和[生产环境上线](/api-wen-dang/readme-1/production-go-live.md)。

#### 集成期间的通知设置

使用[Webhook 概述](/api-wen-dang/product-guides/extensions-and-integrations/webhook-overview.md)了解 webhook 事件覆盖范围和处理指导。

使用[多渠道通知](/api-wen-dang/product-guides/extensions-and-integrations/multi-channel-notifications.md)进行 webhook、邮件、钉钉、企业微信、Slack 和 Teams 的渠道设置。

#### 需要选择预订流程？

使用[产品指南](/api-wen-dang/product-guides.md)。

当您在三种主要预订路径之间做决策时，从[预订概述](/api-wen-dang/product-guides/booking/booking-overview.md)开始。
