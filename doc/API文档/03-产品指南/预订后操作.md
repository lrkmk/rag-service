* 选择合适的预订后工作流
* 判断是退款、作废、附加服务、维护还是 PNR 问题
* 安排出票后的运营跟进

### 常见问题

#### 预订流程完成后应该使用什么？

使用此部分处理退款、作废、出票后附加服务、订单维护和 PNR 认领工作流。

先检查当前订单状态，然后再执行预订后操作。

#### 应该选择哪个预订后路径？

使用**退款**进行退款报价、提交和状态跟踪。

当订单仍在航司作废窗口内时，使用[**作废**](/api-wen-dang/product-guides/post-booking/void.md)，并遵循专门的作废流程。

使用**出票后附加服务**处理出票后的行李操作。

当预订需要运营跟进或干预时，使用**订单维护**和**PNR 认领与提取**。

### 建议检查顺序

1. 确认最新的预订状态。
2. 判断是退款、作废、附加服务、维护还是 PNR 相关问题。
3. 使用对应的工作流页面。
4. 根据需要与 webhook、订单查询或运营跟进进行核对。
