# 309 vs 409

当附加服务选择因当前 `productCode` 可能过时或映射到错误的航段而失败时，使用本页面。

### 简要回答

`309` 表示附加服务 `productCode` 无效或过时。

`409` 表示行李 `productCode` 与目标航班航段不匹配。

两者都是附加服务映射问题。

都不应在未更改的情况下重试。

### 常见问题

#### `309` 通常意味着什么？

意味着所选附加服务在当前预订上下文中不再有效。

使用最新的附加服务响应并重建请求。

#### `409` 通常意味着什么？

意味着行李产品映射到了错误的航段。

再次检查航段级别的行李映射。

#### 新的 `sessionId` 或 `OfferId` 会影响附加服务代码吗？

是的。

新的预订上下文可能会使旧的附加服务代码过时。

### 核心区别

#### `309` — 附加服务未找到

**通常含义**

所选附加服务代码对当前预订上下文无效。

**典型原因**

* 新的验证后重复使用旧的 `productCode`
* 新的报价查询后重复使用旧的 `productCode`
* 行程或乘客组合发生变化
* 附加服务数据缓存时间过长

**下一步操作**

从当前上下文刷新附加服务。

然后仅使用返回的 `productCode` 值。

#### `409` — 行李代码与航段不匹配

**通常含义**

行李 `productCode` 不属于订单请求中的目标航班航段。

**典型原因**

* 航段映射错误
* 联程航段的行李分配不一致
* 来自一个航段的行李代码用于另一个航段

**下一步操作**

重新检查航段映射。

然后为正确的航段结构重建行李选择。

### 快速决策规则

在生产中使用此规则：

* `309` — 刷新附加服务上下文
* `409` — 修复行李到航段的映射

### 首先检查什么

当这些错误出现时，检查：

* 最新的 `sessionId` 或 `OfferId`
* 最新的行李或座位响应
* 预订请求中的航段结构
* 联程航段是否在每个方向使用一致的行李选择

### 常见错误

#### 在新的验证后重复使用附加服务

不要这样做。

新的 `sessionId` 可能会使旧的附加服务上下文失效。

#### 在不同航段间重复使用行李代码

不要这样做。

一个航段的有效行李代码可能对另一个航段无效。

#### 将 `409` 视为通用过时代码错误

不要这样做。

这通常是航段映射问题。

### 最佳实践

保持附加服务查询靠近 `order.do`。

如果预订上下文发生变更，在构建订单前重新运行附加服务查询。
