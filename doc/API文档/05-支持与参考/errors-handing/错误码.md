使用本页面决定是重试、从更早的步骤重新开始，还是在再次调用支付前停止。

当你需要以下内容时，从这里开始：

* 确定错误代码通常意味着什么
* 决定是否安全地重试请求
* 快速跳转到正确的故障排除页面

大多数团队应首先确定失败的步骤。

然后判断问题是由输入、标识符过期、库存变化、支付状态还是平台条件引起的。

### 快速答案

#### 我应该重试这个 Atlas API 错误吗？

仅当失败明显是临时性的时候才重试。

超时、短暂的负载和一些航空公司侧的故障通常是可重试的。

输入错误、过期的标识符、过时的附加服务和重复支付风险是不可重试的。

#### 哪些错误通常需要新的标识符？

诸如 `202`、`206`、`207`、`210`、`301`、`302`、`308` 和 `631` 等代码通常意味着当前的预订上下文不再有效。

从正确的更早步骤重新开始，而不是重新提交相同的标识符。

#### 何时应避免重试 `pay.do`？

当订单可能已经支付、正在出票或等待航空公司确认时，不要重试支付。

对于 `402`、`404`、`406` 和 `615` 等情况，首先检查订单状态。

#### `getOfferPrice.do` 订单有什么变化？

在执行流程中更积极地处理支付和出票失败。

活动窗口为 5 分钟，而不是标准的 30 分钟保留模式。

仅在确认订单仍然可支付且仍在活动窗口内时重试。

### 429 请求限制处理

Atlas 可能会用 Atlas 错误代码 `429` 拒绝超限的预订前请求。

此策略适用于以下池：

* `search.do` 使用 `10 QPS`
* `verify.do` 和 `getOffers.do` 共享 `60 QPM`
* `seatAvailability.do` 和 `getLuggage.do` 共享 `60 QPM`

`order.do` 和 `pay.do` 不在此策略范围内。

#### 响应字段

当返回 `429` 时，使用这些字段：

* `code` — 错误代码
* `message` — 错误摘要
* `limitType` — 活动的限制桶
* `limit` — 当前配置的限制
* `retryAfter` — 重试前的等待时间（秒）

#### 响应示例

```json
{
  "code": 429,
  "message": "Rate limit exceeded. Please retry after the current window resets",
  "limitType": "SEARCH_QPS",
  "limit": 1,
  "retryAfter": 1
}
```

使用 [API 请求限制](/api-wen-dang/product-guides/booking/booking-overview/api-request-limits.md) 获取完整策略、计数规则和池映射。

#### 什么会计入限制

这些请求会计入：

* 成功的请求
* 无结果的请求
* 业务失败和航空公司失败
* 缓存命中响应

这些请求不计入：

* 已被请求限制治理拒绝的请求
* 认证失败
* 参数验证失败
* 权限或安全策略拒绝
* 幂等性阻止的重复请求（从未进入业务处理）

### 重试策略

在任何重试前使用此规则：

* **是**：现在或短暂延迟后重试
* **部分**：仅在更改条件后重试
* **否**：先修复输入、流程或账户设置

典型的"部分"案例：

* `429`：等待 `retryAfter`，然后以较低频率重试
* `110`：在重试前增加退避
* `406`：等待正在进行的支付完成
* `415`：先完成 `orderCommit.do`
* `616` 或 `633`：切换支付方式
* `108`、`113`、`324`：稍后重试，不要立即重试

{% hint style="warning" %}
如果支付可能已经开始或完成，在任何重试前查询订单状态。
{% endhint %}

{% hint style="warning" %}
对于 `getOfferPrice.do` 订单，不要使用通用重试循环。

一旦执行窗口几乎耗尽，停止重试。
{% endhint %}

| ----- | ----------------------------- | ---- | ---------------------------------- |
| `429` | 请求限制已超出                       | 部分   | 等待 `retryAfter` 并降低请求频率            |
| `107` | 余额低于允许的阈值                     | 否    | 充值账户，然后重试                          |
| `108` | 航线暂时受限或关闭                     | 部分   | 稍后重试，不要立即重试                        |
| `110` | 搜索 QPS 限制命中                   | 部分   | 增加指数退避，然后重试                        |
| `112` | 搜索超时                          | 是    | 短暂等待后重试                            |
| `123` | 付费订单的搜索量过高                    | 否    | 减少不必要的搜索流量                         |
| `126` | 智能搜索 `requestId` 无效或过期        | 否    | 开始新的智能搜索                           |
| `127` | 搜索超时且请求已结束                    | 是    | 运行新的搜索                             |
| `202` | `routingIdentifier` 在 6 小时后过期 | 否    | 重新运行搜索并使用新的标识符                     |
| `205` | 验证超时                          | 是    | 短暂等待后重试验证                          |
| `206` | 搜索后航班不再可用                     | 否    | 从搜索重新开始                            |

| `324` | 订单期间航空公司系统问题                  | 部分   | 稍后重试，如果重复则联系运营团队                   |
| `406` | 支付已在进行中                       | 部分   | 在任何重试前等待并查询订单状态                    |

| `411` | 由于余额或账户检查导致支付失败               | 否    | 首先验证余额和支付输入                        |

| `616` | 卡需要 3DS                       | 是    | 使用非 3DS 卡或存款模式                     |
| `617` | 出票时存款余额过低                     | 否    | 重试前充值余额                            |

| `809` | 出票完成前尝试退款                     | 否    | 等待出票完成                             |
| `811` | 退款请求缺少必需的行程航段                 | 否    | 提交完整行程                             |

不要将每个错误都视为可重试的。

有些错误需要新的标识符。\

有些绝不能触发另一次支付调用。

### 高风险错误

这些情况导致最多的可避免的重复失败：

#### `110` 并发请求过多

不要立即重试。

增加指数退避。\
从约 `1s` 开始。\
上限约 `30s`。

#### `429` 请求限制已超出

不要立即重试。

首先等待返回的 `retryAfter` 值。
