# Atlas废票政策说明

废票（Void）是指客户在出票后，在航司允许的废票窗口期内，对符合条件的客票发起作废处理。废票与退票为两类独立的售后处理类型，Atlas 将废票作为独立售后服务进行管理，不并入退票或特殊退票范畴。

Atlas 提供废票申请的系统自动化受理、提交执行及结果同步服务。最终能否废票成功，以航司规则、废票窗口期、订单状态及系统返回结果为准。

### 一、受理范围

Atlas 当前仅支持符合条件的订单通过系统发起废票申请。订单需同时满足以下条件：

* 订单通过 Atlas 出票；
* 对应航司已支持废票服务；
* 废票申请在系统允许的废票窗口期内；
* 订单状态符合废票处理条件；
* 系统支持对该订单进行自动化提交和结果返回。

为降低因临近航司截止时间导致废票失败的风险，Atlas 会根据航司废票规则设置对应的系统废票窗口期：

* 对于航司规定**出票后 24 小时内可废票**的订单，Atlas 废票窗口期设置为**出票后 23 小时内**；
* 对于航司规定**仅出票当天可废票**的订单，Atlas 废票窗口期设置**为出票当天 23:30 前**。

Atlas 废票服务将根据航司能力及系统支持情况逐步开放。具体是否支持废票，以 ATRIP 页面或 API 返回结果为准。

暂不支持 PNR 分离后废票、部分旅客废票、部分航段废票，以及需要人工线下处理的废票场景。

### 二、处理原则

{% step %}

#### 7×24 小时自动化受理

系统支持全天候自动化受理，客户可在符合系统废票窗口期及相关条件时随时发起申请。
{% endstep %}

{% step %}

#### 仅支持系统自动化处理，不提供人工干预

由于废票对时效要求较高，当前废票申请仅通过系统自动化方式提交和执行，不提供人工废票或人工兜底处理。若航司、订单或具体场景暂不支持自动化废票，Atlas 暂无法受理该废票申请。
{% endstep %}

{% step %}

#### 执行失败即时通知

如系统自动执行失败，Atlas将在**5分钟内**向客户同步失败结果，并结束本次废票流程。客户可根据订单实际情况，选择是否通过其他可用渠道进一步处理。
{% endstep %}
{% endstepper %}

### 三、废票结果与退款说明

废票申请提交成功，不代表最终一定废票成功，也不代表客户一定能获得退款。废票可能因航司规则限制、超出废票窗口期、订单状态异常、航司系统返回失败等原因无法完成。

最终废票结果、退款状态、退款金额及退款时间，均以航司规则、系统返回结果及 Atlas 对账确认流程为准。废票成功后，不代表可立即收到退款；实际退款时间仍取决于航司退款周期、资金返回时间及 Atlas 对账确认流程。
