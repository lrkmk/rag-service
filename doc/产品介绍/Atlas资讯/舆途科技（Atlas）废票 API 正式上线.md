# 舆途科技（Atlas）废票 API 正式上线

对于机票代理而言，废票（VOID）一直是一项对时间高度敏感的售后操作。

每一家航空公司都有严格的废票时间窗口：有的要求出票后 24 小时内完成，有的甚至只允许在出票当天操作。一旦错过截止时间，原本可以直接作废的机票可能需要进入复杂的退票流程，不仅增加运营成本，也影响最终客户体验。

**舆途科技（Atlas）废票服务现已通过 API 正式上线**，扩展的售后服务能力。&#x20;

机票代理现可通过 API **7×24 小时自动提交、管理和追踪废票请求**，减少人工操作，避免错过航司废票窗口。&#x20;

针对廉航售后场景对时效性和自动化运营的诉求，废票 API 可帮助机票代理降低运营成本、提升客户体验，并更高效地管理预订后服务。&#x20;

📌 废票 API 默认开放给所有舆途科技（Atlas） 客户，无需额外申请开通权限，仅需完成 API 对接即可使用。&#x20;

&#x20;

### <mark style="color:$primary;">废票 API 将为您的业务带来什么价值？</mark>

#### 一、7×24 小时自动提交废票请求

系统自动识别并提交符合条件的废票请求，无论白天、夜晚还是节假日，均可持续运行。&#x20;

无需人工登录航空公司系统，无需发送邮件，也无需依赖运营人员手动处理，让废票真正实现自动化。&#x20;

#### 二、更快获得处理结果，避免错过最佳处理时机&#x20;

过去，人工废票通常需要平均约 4 小时完成处理。&#x20;

通过废票 API，您可以在 **5 分钟内收到处理成功或失败的反馈**。即使无法完成废票，也能够第一时间获得通知，及时采取其他售后措施，最大程度降低损失。 &#x20;

#### 三、全流程可追踪&#x20;

通过 API 响应、系统通知和运营报告追踪每一笔废票请求。从提交到结果确认，**全程状态透明可见**。&#x20;

#### 四、降低运营成本，让团队专注更高价值工作&#x20;

自动化意味着更少的人工接触点、更低的操作成本和更高的处理效率。&#x20;

您的运营团队可以从重复性的废票操作中解放出来，将更多精力投入客户服务、异常订单处理以及业务增长。 &#x20;

#### 五、更快速、更流畅的客户体验&#x20;

当客户需要取消订单时，等待越短，体验越好。&#x20;

通过自动化废票能力，客户可以更快获得处理结果，减少等待和不确定性，提升整体服务体验。

#### 六、内置安全缓冲机制，进一步提升废票成功率&#x20;

考虑到网络波动、系统延迟以及临近截止时间操作等风险，我们在部分航司废票规则基础上设置了安全缓冲时间，提前保护您的订单。&#x20;

例如：&#x20;

* 航司允许出票后 24 小时内废票 → 我们将内部截止时间设置为出票后 23 小时&#x20;
* 航司仅允许出票当天废票 → 我们 将内部截止时间设置为当天 23:30 前&#x20;

### 常问问题

<details>

<summary>废票是否一定成功？</summary>

不一定。是否符合废票条件以及最终是否批准，由对应航司决定。舆途科技（Atlas）负责自动提交请求、追踪状态并及时通知最终结果。

</details>

<details>

<summary>什么时候会收取废票服务费？</summary>

仅当航司成功批准并完成废票请求后，才会收取废票服务费。如失败，则不会收取任何服务费。服务费按月度结算。

</details>

<details>

<summary> PNR分离的订单能废吗？</summary>

暂时不能。后续如支持范围有调整，我们将及时同步通知。

</details>

<details>

<summary>废票可否退税？</summary>

不含税。（即废票退款金额不包含税费部分

</details>

<details>

<summary>如何查询订单是否还能废票？</summary>

请调用 voidQuotation.do，通过 orderNo 获取该订单最新废票条件、窗口期、报价及 voidOfferId。是否可废票以接口返回结果为准。

</details>

<details>

<summary>除了调用 voidQuotation.do 之外，有没有更简单或更直观的方法来查询废票条件？</summary>

目前 voidQuotation.do 是最准确的方法。

它是订单维度的实时判断接口，能直接返回该订单当前时刻的真实可废票状态（isVoidable）和对应的报价ID（voidOfferId），避免了因缓存或静态规则导致的信息滞后或偏差。

</details>
