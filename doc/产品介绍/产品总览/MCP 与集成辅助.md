# MCP 与集成辅助

MCP 帮助开发者更快理解文档和调用流程。

Skill 帮助团队按标准方式完成集成和交付。

### GitBook MCP 能做什么

#### 文档智能检索

用自然语言查询 API 文档、产品说明和帮助内容。

适合快速查参数、错误码和业务规则。

#### API 流程导航

按业务链路给出下一步建议。

适合梳理搜索、验价、下单和支付的调用顺序。

#### 相似 API 对比

对比功能接近的接口。

适合做接口选型和边界判断。

#### 错误诊断与定位

根据错误信息给出排查方向。

适合缩短联调和支付问题定位时间。

#### 沙箱联调辅助

在沙箱环境中提供实时指引和测试建议。

适合首次联调和异常场景验证。

### 推荐使用方式

#### 项目启动阶段

先用 MCP 快速理解业务链路和接口边界。

适合梳理搜索、验价、下单、支付和售后的完整流程。

#### 联调阶段

用 MCP 查询参数、错误码和典型处理方式。

适合边开发边定位问题。

#### 问题排查阶段

用 MCP 缩小排查范围，再回到真实请求验证结论。

适合处理支付失败、状态异常和字段理解偏差。

### 集成辅助 Skill 包含什么

* 标准化对接流程
* 项目启动清单

### 启用 MCP 前需要确认什么

* 使用的 AI 工具支持 MCP
* GitBook 账号可访问 Atlas 文档空间
* 授权时使用同一个 GitBook 账号
* 账号权限完整

MCP 服务地址：`https://resources.atriptech.com/~gitbook/mcp`

### 适合优先用 MCP 处理的问题

* 某个接口该在流程里的哪一步调用
* 两个相似接口该怎么选
* 某个错误码通常意味着什么
* 某个字段在什么场景下出现

### 不适合只靠 MCP 解决的问题

* 最终返回字段是否与当前环境一致
* Webhook 实际推送内容是否已变化
* 支付成功后航司侧是否最终出票
* 沙箱与生产的行为是否完全一致

### 最佳实践

* MCP 和 Skill 配合使用
* 先让 MCP 辅助理解流程，再用沙箱实呼验证
* 对关键字段、Webhook 和支付结果做真实联调

### 上线前建议检查

* 已用真实请求验证关键字段
* 已验证支付成功、失败和重试路径
* 已验证 Webhook 接收与兼容处理
* 已确认团队理解主链路和售后边界

### 使用边界

MCP 适合辅助开发。

它不能替代真实联调和结果验证。

以下内容必须以实际结果为准：

* API 响应字段与状态
* Webhook 接收结构
* 支付成功与失败路径
* 沙箱与生产环境差异

黄金原则：AI 给出的建议，必须经过真实请求验证。
