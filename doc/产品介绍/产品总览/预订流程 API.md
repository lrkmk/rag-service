# 预订流程 API

Atlas 提供完整的预订流程 API。

标准链路从搜索开始，以支付和出票结束。

{% stepper %}
{% step %}

### 搜索航班

使用搜索接口查询可售航班、票价和库存信息。

支持单程、往返、多程和开口程。

搜索结果可返回：

* 航班基础信息
* 票价与税费
* 行李额度
* 退改签摘要
* Fare Family 信息
  {% endstep %}

{% step %}

### 验证票价

下单前必须执行验价。

可使用 `verify.do` 或 `getOffers.do` 刷新价格和库存。

验价用于确认：

* 实时价格
* 座位库存
* 行李选项
* 详细规则
  {% endstep %}

{% step %}

### 创建并确认订单

通过验价会话创建订单。

订单创建后，系统返回订单号、状态、支付时限和应付金额。

订单确认阶段适合做乘客信息、附加服务和支付方式校验。
{% endstep %}

{% step %}

### 支付与出票

使用 `pay.do` 完成支付并触发出票。

支持预存款、VCC 透传，以及失败后的重试策略。

典型目标：

* 24/7 自动履约
* 5 分钟内完成大部分订单出票
* 失败时快速告警
  {% endstep %}
  {% endstepper %}

### 搜索能力

#### 基础搜索

适合常规机票查询场景。

支持城市代码、机场代码、出行日期、乘客类型和舱位偏好。

#### 高级搜索

适合做更精准的结果推荐。

可结合灵活日期、相似航线和中转方案优化展示。

#### Fare Family

同一航班可返回多个运价产品。

适合展示基础票价、标准票价、灵活票价和商务票价。

### 核心接口

#### `search.do`

用于查询实时可售航班和票价。

适合做首轮结果返回和航班列表展示。

#### `verify.do`

用于下单前刷新价格和库存。

适合确认搜索结果是否仍然可订。

#### `getOffers.do`

用于获取更完整的报价和规则信息。

适合复杂行程、多航司组合和需要展示详细规则的场景。

#### 订单创建接口

用于基于验价结果创建订单。

创建后会返回订单号、状态、支付时限和金额信息。

#### `pay.do`

用于完成支付并触发出票流程。

适合结合预存款、VCC 和支付重试策略使用。

### 推荐调用顺序

`search.do → verify.do / getOffers.do → 创建订单 → pay.do → 查询订单状态`

### 常见场景

#### 只做价格展示

可先使用 `search.do` 返回列表。

进入下单前，再执行验价。

#### 需要展示详细规则

建议在关键选择节点使用 `getOffers.do`。

这样更适合展示复杂规则、行李和退改签细节。

#### 需要控制价格风险

必须在下单前强制执行验价。

这样可以减少搜索价和成交价之间的差异。

### 附加服务

Atlas 支持在预订流程中销售附加服务。

典型能力包括：

* 行李加购
* 座位图与选座
* 出票后继续购买部分服务

### 关键规则

* 搜索结果仅供参考
* 创建订单前必须完成验价
* 支付结果应结合订单状态和后续通知确认
* 复杂场景建议先在沙箱完整走通流程

{% hint style="warning" %}
如果未执行 `verify.do` 或 `getOffers.do`，系统不应直接进入创建订单流程。
{% endhint %}
