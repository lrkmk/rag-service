# 退款、查询与预订后错误

当出票后发生问题时，使用本页面。

当你需要以下内容时，从这里开始：

* 了解预订后的退款、作废、订单查询或附加服务失败
* 决定是等待、重试还是先修复请求
* 将案例路由到正确的预订后工作流

### 常见问题

#### 哪些预订后错误通常意味着我们应该先等待？

诸如 `809`、`814` 和 `117` 等代码通常意味着订单或退款流程尚未准备好进行下一步。

等待出票或正在进行的提交完成后重试。

#### 哪些预订后错误通常意味着我们不应重新提交？

诸如 `816`、`817`、`818`、`504` 和 `505` 等代码通常意味着退款或行李操作已存在或已经应用。

查询当前状态而不是再次提交。

#### 哪些错误最特定于专用的作废流程？

诸如 `822` 和 `823` 等代码指向仅作废的提交问题。

当订单仍在航空公司作废窗口内时，使用专用的[作废](/api-wen-dang/product-guides/post-booking/void.md)流程。

### 退款报价和提交

#### `801` 订单不存在

订单号错误。

**操作**

* 检查原始机票订单号

#### `809` 订单未出票

在出票完成前请求退款。

**操作**

* 等待出票完成
* 稍后重试退款

#### `811` 未输入完整的行程

退款请求不包括所需的完整行程。

**典型原因**

* 缺少联程的一个航段
* 必须一起处理的往返行程的部分退款
* 已飞航段

**操作**

* 提交完整的所需行程
* 对于部分已飞情况，联系航空公司或 Atlas

#### `814` 退款提交正在进行中

先前的退款提交仍在处理中。

**操作**

* 等待后重试

#### `816` / `817` / `818` 退款已提交

匹配的退款或索赔已存在。

**操作**

* 不要重新提交
* 查询退款状态

### 作废流程

作废使用专用端点，应遵循作废报价 -> 作废提交 -> 作废查询的顺序。

如果订单仍在航空公司作废窗口内，使用[作废](/api-wen-dang/product-guides/post-booking/void.md)而不是退款流程。

#### `822` 超过机票作废截止时间

作废请求在航空公司作废窗口关闭后提交。

Atlas 立即拒绝此请求。

典型消息：

* `作废截止时间已过。此机票不能再作废`

**操作**

* 不要重试相同的作废提交
* 重新检查是否应转为退款流程

#### `823` 未包含所有乘客的作废

作废请求未覆盖所需的乘客范围。

Atlas 仅接受整单作废。

**操作**

* 不要仅为部分乘客提交作废
* 仅在请求覆盖整单时提交作废

### 查询订单

#### `703` 未找到订单

查询参数与现有订单不匹配。

**操作**

* 重新检查 `orderNo`、`airlinePNR` 和承运人值

#### `705` 超时

订单检索请求超时。

**操作**

* 短暂等待后重试

### 预订后附加服务

#### `117` 订单未处于已出票状态

附加服务搜索尝试过早。

**操作**

* 等待原始订单出票

#### `118` 此航空公司不支持附加服务

Atlas 无法为此航空公司支持此附加服务流程。

**操作**

* 检查直接的航空公司流程或创建服务请求

#### `502` 附加服务订单期间价格变动

搜索后附加服务价格发生变化。

**操作**

* 重新搜索附加服务选项

#### `504` / `505` 不允许添加额外行李

行李已在预订流程或先前的预订后订单中添加。

**操作**

* 不要提交重复的行李订单
