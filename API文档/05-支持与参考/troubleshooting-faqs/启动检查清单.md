### 常见问题

#### 什么是启动检查清单？

此清单涵盖在开始开发前应明确的架构、流程和集成设计决策。

### 流程选择

* [ ] 选择主要预订流程：标准搜索 vs. Get Offer vs. 两者
* [ ] 确认是否将实施智能搜索
* [ ] 决定是否将支持座位和行李附加服务
* [ ] 计划如何满足搜索并发限制

### 支付设计

* [ ] 确认支付模式：存款、VCC 直通或两者
* [ ] 计划如何处理拆分订单支付（往返 / 多订单场景）
* [ ] 确认 VCC 卡数据要求（持卡人地址字段）
* [ ] 计划如何处理 3DS 卡（回退到非 3DS 卡或存款模式）

### 出票处理

* [ ] 计划如何在 `pay.do` 之后轮询最终出票状态
* [ ] 决定如何处理延长的出票窗口（最长 1 小时）
* [ ] 确认如何监控和响应未出票订单（升级路径）

### 预订后

* [ ] 决定是否将支持退款
* [ ] 决定是否将支持作废
* [ ] 决定是否将支持出票后附加服务

### 发布范围

* [ ] 确定上线时的目标航空公司
* [ ] 计划沙箱测试范围
* [ ] 计划 UAT 范围
* [ ] 确认结算和对账流程
