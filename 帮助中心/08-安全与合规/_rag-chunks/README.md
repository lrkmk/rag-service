## 切分依据

1. **文章为最小一级边界**：2 篇文章互不跨界拼接，各自独立产出 1 个 parent + N 个 child。
2. **规则单元整段保留**：按源文档的功能小标题（访问密钥、SSO、密码策略、MFA、IP白名单、审核日志等）切分，不按字数机械切断。例如 `aq-c08`（密码策略参数）把长度/复杂度/过期/历史/锁定五组默认值+可调范围作为一个不可再分的参数集合保留在同一个 chunk 里，避免拆开后丢失"某项默认值属于哪个参数"的对应关系。
3. **权限限制单独打标签**：涉及"仅管理员可操作"这类访问限制的内容单独切成 `rule_type: 例外` 的 chunk（如 `aq-c04`、`aq-c12`），便于检索时区分"功能怎么用"与"谁能用"。
4. **Parent-Child 结构**：每个 child 都有 `parent_id`，可回带 parent 摘要帮助判断规则出自哪篇文章。
5. **FAQ**：本分类源文件均为说明文档，无 FAQ 形式内容，未涉及。
6. **Metadata 字段**：与退票示例一致，含 `level1_category`（安全与合规）、`level2_category`（按文章区分）、`applicable_carrier`（本分类下均为"通用"，不涉及航司差异）、`updated_at`、`source_path`。
7. **数字类规则未抽表**：本分类的数字规则（如密码策略默认值、IP白名单条目上限50个、审核日志留存1年/单次下载1个月）数量少、结构简单，未构成大型表格，因此直接保留在对应文字 chunk 中，未做结构化表抽取。

## 判断记录

- **`updated_at` 全部为 `null`**：源目录下的导航文件 `安全与合规.md` 未像「退票」分类的 `退票.md` 那样标注"最后更新日期"，因此本分类所有 parent/child 的 `updated_at` 均记为 `null`，而非编造日期。
- **`applicable_carrier` 全部为"通用"**：两篇源文章均为平台级账户/安全功能说明，不涉及具体航司差异。
- 与 `09-Atlas功能/Atlas Security and Compliance.md` 的重叠评估见该目录 README；`安全与合规说明.md` 是操作导览版本（含具体 UI 路径、按钮名称、参数默认值），两者内容互补而非重复，因此分别在各自分类下保留为独立 parent 记录，未合并。
