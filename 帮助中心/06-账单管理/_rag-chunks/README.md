# 账单管理 / Invoice管理 — RAG 切片

对「06-账单管理」下 1 篇源文章（`Invoice管理.md`）做的切片，沿用「04-售后票务/退票」示例的格式与原则。`账单管理.md` 是导航/链接页，未纳入切片。

## 分类结构说明

06-账单管理 是顶级分类，「Invoice管理」下没有更细的子分类，因此本文章的 `level1_category` 与 `level2_category` 均取值 `账单管理` / `Invoice管理` 本身（与售后票务示例中"大类/子类"两层结构不同）。

## 文件

- `parents.jsonl` — 文章级 parent 记录（1 条）
- `children.jsonl` — 规则单元级 child 记录（6 条）

## 切分依据

1. **文章为最小一级边界**：全篇只有 1 篇源文章，产出 1 个 parent + 6 个 child。
2. **按功能小节切分，不按字数机械切断**：源文章的一/二/三/四级标题（功能概述、操作信息、发票配置说明、账单核心字段、账单汇总字段）各自成为独立或多个 child 的边界。
3. **字段词典整体保留**：账单核心字段表（26 个字段）与账单汇总字段表（17 个字段，含 Usage、期末余额两条计算公式）分别整体保留为 `inv-c04`、`inv-c06` 两个 child，未按行/按字数切断字段表——计算公式与其相关字段定义（如 Usage 与其五项构成字段）保持在同一个 chunk 内，不拆开条件与结果。
4. **例外/免责说明单独打标签**："重要说明"三条（空字段属正常现象、非预存款支付场景下金额口径、统一 GMT+8 时区）与字段表本身在语义上不同，单独切为 `inv-c05`，`rule_type` 标记为"例外/计算规则"。
5. **航司特例**：本文无航司特例内容（"Airline"仅为账单字段名，非规则本身），`applicable_carrier` 统一为 `通用`。
6. **Parent-Child 结构**：每个 child 都带 `parent_id`，可回溯到「Invoice管理」文章摘要。
7. **Metadata 字段**：与退票示例字段结构完全一致。源文章及其导航页 `账单管理.md` 均未标注更新日期，`updated_at` 统一记为 `null`。
8. **FAQ / 数字类大表抽取**：本文不含 FAQ；两张字段表虽然较长，但属于"字段名-含义"词典类内容而非可精确查表的数值型规则表（不同于退票时限表的 131 航司数值矩阵），因此未额外抽取为独立结构化 JSON，仍以语义 chunk 形式呈现，便于按字段名检索命中释义。
