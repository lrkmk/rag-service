# 工单子分类 — RAG 切片

对「04-售后票务/工单」下唯一源文章做的切片，沿用「退票」示范的切法与 schema。

## 文件

- `parents.jsonl` — 文章级 parent 记录（1 条），含一句话摘要 + child_chunk_ids
- `children.jsonl` — 规则单元级 child 记录（6 条）

## 切分说明

1. **文章为最小一级边界**：本子类仅 1 篇源文章 `工单使用说明.md`，产出 1 个 parent + 6 个 child。
2. **规则单元整段保留**：按原文一至四级标题切分为“功能简介 / 提交入口（订单类+非订单类合并为一个 chunk，因二者是并列对照关系）/ 创建工单入口触发条件 / 提交步骤（四个子步骤合并为一个 chunk，因步骤之间强依赖顺序、拆开会破坏操作连贯性）/ 查看已提交工单 / 工单状态说明（四种状态含流转规则，整体保留）”，未按字数机械切断。
3. **航司特例**：本文不涉及具体航司差异，全部 child 的 `applicable_carrier` 为 `通用`。
4. **Parent-Child 结构**：每个 child 都带 `parent_id`，可回带 parent 摘要，帮助模型判断规则出自「工单」而非「退票/改期」等其他售后子类下同样涉及“提交申请”的通用流程说明。
5. **Metadata 字段**：与退票示例一致，含 `level1_category`（售后票务）、`level2_category`（工单）、`applicable_carrier`、`updated_at`（源文章及其上级导航文件均未标注“最后更新日期”，故记为 `null`）、`source_path`。

## 未覆盖范围

这只是「工单」一个子类（且仅 1 篇源文章）的切片示范。若后续新增工单相关的 FAQ 或专项说明文章，可按同样规则追加 parent/child 记录。
