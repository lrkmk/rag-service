# 特殊航司子分类 — RAG 切片

对「04-售后票务/特殊航司」下 2 篇源文章做的切片，沿用「退票」示范的切法与 schema。

## 文件

- `parents.jsonl` — 文章级 parent 记录（2 条），每条含一句话摘要 + child_chunk_ids
- `children.jsonl` — 规则单元级 child 记录（5 条）

## 切分说明

1. **文章为最小一级边界**：2 篇文章各自独立产出 1 个 parent + N 个 child。
2. **规则单元整段保留**：`Wizz 航司售后处理说明` 按原文三级标题切（一、售后操作处理规则 / 二、退款形式 / 二、不同支付方式的后续处理 / 三、重要说明），其中“退票及退款处理说明”一节因内容较长且包含两类不同规则（先转积分再转现金的退款形式、VCC与余额两种支付方式的后续差异处理），拆成 `w6-c02`/`w6-c03` 两个 chunk，但每个 chunk 内部的条件+结果仍保持完整未再切分。`如何加购婴童票？` 全文仅一段规则说明，未做进一步拆分。
3. **航司特例单独打标签**：`Wizz 航司售后处理说明` 一文开篇明确写明“适用于 Wizz 相关航司订单，包括 W4、W6、W9”，因此其全部 child 的 `applicable_carrier` 标注为 `W4/W6/W9`（而非仅 `W6`），以准确反映源文实际覆盖的三个航司代码——Wizz Air Malta（W4）、Wizz Air Hungary（W6）、Wizz Air Abu Dhabi（W9）。`如何加购婴童票？` 一文内容是系统功能限制说明，未提及任何具体航司差异，标注为 `通用`。
4. **Parent-Child 结构**：每个 child 都带 `parent_id`，可回带 parent 摘要用于判断规则出自「Wizz 专属规则」还是「通用婴童票加购限制」——两者虽同属「特殊航司」子类，但适用范围完全不同。
5. **Metadata 字段**：与退票示例一致，含 `level1_category`（售后票务）、`level2_category`（特殊航司）、`applicable_carrier`、`updated_at`（两篇源文章及其上级导航文件均未标注“最后更新日期”，故记为 `null`）、`source_path`。

## 判断说明（Wizz 航司标签）

任务要求中给出的示例标签是 `W6`，但源文件正文第一句即声明适用范围为“W4、W6、W9”三个航司代码，且后续条款（售后处理规则、退款处理流程、重要说明）均未区分三者的差异，是作为同一套规则统一适用。为遵守“不臆造事实、不为求简化而丢失精确度”的原则，本次切片将 `applicable_carrier` 标注为 `W4/W6/W9`，而非仅 `W6`，避免检索时因标签过窄而漏检 W4、W9 相关问题。若后续需要单独区分三者的细颗粒度差异，需要更详细的源资料支持。

## 未覆盖范围

这只是「特殊航司」一个子类的切片，未覆盖其余售后子类或该子类之外的其他特殊航司说明文档（若后续新增更多航司专属文章，可按同样规则追加）。
